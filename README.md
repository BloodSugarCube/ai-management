# AI-management

Веб-приложение на **Laravel 9** (совместим с **PHP 8.0+**) для учёта сотрудников и задач из **Redmine**, подбора исполнителя с помощью **GenAPI** и ручного редактирования профилей сотрудников.

## Возможности

- **Админ-панель** с входом по логину/паролю из `.env` (поддерживается `ADMIN_PASSWORD_HASH` на bcrypt и ограничение частоты попыток входа).
- **Сотрудники**: список из Redmine, текущие задачи в статусах «in progress» (настраивается), поля грейдов/компетенций/опыта, сумма запланированных часов по невыполненным назначенным задачам, сохранение в MySQL.
- **Задачи**: таблица с сортировкой по столбцам; по умолчанию выше задачи **без исполнителя**; раскрываемое описание; кнопка «Подобрать исполнителя».
- **Автоназначение** (переключатель, по умолчанию включён): один клик — два запроса к GenAPI, затем назначение лучшего кандидата в Redmine.
- **Ручной режим**: модальное окно с до **5** кандидатами (логин, процент, пояснения), одиночный выбор через чекбоксы, сохранение и закрытие по кнопке, крестику или клику по фону.
- **Интеграции** (по ТЗ):
  - Redmine: `app/Components/RocketChat` (HTTP REST).
  - GenAPI: `app/Components/GenApi` (POST к сети + синхронное ожидание результата через polling, **без webhook**).
- **Сервис рекомендаций** `TaskAssigneeRecommendationService`: два последовательных запроса в GenAPI (см. `ARCHITECTURE.md`).
- **Команды**:
  - `php artisan redmine:sync-employees`
  - `php artisan redmine:sync-tasks`

## Требования

- PHP **8.0.2+** (для Laravel 9)
- Composer
- MySQL

## Установка

```bash
composer install
cp .env.example .env
php artisan key:generate
```

Настройте `.env`: `DB_*`, `REDMINE_*`, `GENAPI_*`, `ADMIN_*`.

Сгенерируйте bcrypt-хэш пароля администратора:

```bash
php artisan admin:hash-password "your-strong-password"
```

Скопируйте вывод в `ADMIN_PASSWORD_HASH` и задайте `ADMIN_USERNAME`. Для локальной отладки можно временно указать `ADMIN_PASSWORD` (не рекомендуется в продакшене).

Примените миграции:

```bash
php artisan migrate
```

Синхронизация данных из Redmine:

```bash
php artisan redmine:sync-employees
php artisan redmine:sync-tasks
```

Запуск встроенного сервера:

```bash
php artisan serve
```

Фронтенд-сборка (**Vite**): для страниц админки подключаются `resources/css/app.css` и `resources/js/app.js`.

```bash
npm install
npm run dev
```

Для продакшена перед деплоем:

```bash
npm run build
```

Откройте `/login`.

## GenAPI

Укажите `GENAPI_API_KEY`, `GENAPI_STEP1_NETWORK_ID` и (опционально) `GENAPI_STEP2_NETWORK_ID` (если пусто — используется ID первого шага).

Тело запроса по умолчанию содержит поля `is_sync` и `text` с промптом. Дополнительные поля можно передать JSON-строкой в `GENAPI_EXTRA_JSON_BODY`, если ваша сеть GenAPI требует иной формат входа.

Модель должна возвращать **строгий JSON** (см. промпты в `TaskAssigneeRecommendationService`):

1. Шаг 1: `{"candidates":["login1", ...]}` (до 5 логинов).
2. Шаг 2: `{"recommendations":[{"login":"...","match_percent":72.5,"reasons":["..."]}]}`.

При ошибке парсинга или недоступности GenAPI используется упрощённый резервный список кандидатов (по нагрузке).

## Redmine

- Нужен доступ по API-ключу (`REDMINE_API_KEY`) с правами на чтение пользователей и задач и на обновление назначения задачи.
- Метки подтягиваются из пользовательских полей с именами: `Tags`, `Метки`, `Labels`, `Теги` (если есть).
